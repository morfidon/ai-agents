Goal:Shrink what you expose. No edits. No execution.

Method:

Read exports in each module or package.

Map who imports them from outside the module or service.

Flag symbols that have zero external use.

Group findings by file and symbol.

What to Look For:

Unused exports

Classes, functions, types never imported outside their home module.

Over-exposed classes

Many public methods where only a few are used.

Wide DTOs

Data objects with fields no caller reads.

Barrel re-exports

index.ts that exports internals no one needs.

Legacy names kept public

Old API kept for convenience but with no real callers.

Test-only exposure

Public just to satisfy tests. No runtime use.

Transitive leaks

Re-export of a dependency type that callers never touch.

Mixed concerns in public API

Unrelated helpers exported next to the main type.

Heuristics:

Cross-module import scan.

Fan-in at public boundary.

Barrel resolution to real files.

Grep for symbol names in non-local paths.

Optional: git history to see last external use.

Expected Output Format:Readable. One line per finding. Grouped by file and symbol.

File: packages/catalog/src/index.ts
  - Symbol: export * from './internal/PriceRules'
    Exposure: Re-exported but never imported outside catalog
    Severity: Moderate
    Confidence: High

File: src/core/DateUtil.ts
  - Symbol: class DateUtil
    Exposure: 12 public methods, only 3 called outside module
    Severity: Moderate
    Confidence: Medium

File: src/api/dto/OrderDTO.ts
  - Symbol: field discountCode
    Exposure: Present in DTO but no external reader found
    Severity: Moderate
    Confidence: High

File: src/users/public.ts
  - Symbol: export { LegacyUser }
    Exposure: Legacy type has zero external imports in repo
    Severity: Moderate
    Confidence: High

File: packages/ui/src/index.ts
  - Symbol: export { formatMoney }
    Exposure: Helper leaks through UI barrel. No consumer uses it.
    Severity: Moderate
    Confidence: High


Output Rules:

List every public symbol that has no external callers or is wider than needed.

Include file, symbol, short exposure note, Severity, Confidence.

Do not propose refactors.

Sort by path, then symbol.

Use exact names from code.

Severity:

Moderate by default. Public surface bloat slows refactors and increases risk.

Confidence:

High for exports with zero non-local imports.

Medium when only a fraction of methods are used or DTO fields look unused.

Low only if dynamic import patterns hide usage.
