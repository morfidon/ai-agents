# Refactoring Opportunity Planner Agent

## Goal

Pick refactors with the biggest payoff. No edits. No execution.

## Method

- Read results from 3.1 to 3.9.
- Cluster findings by path or module.
- Score overlap and impact.
- Output a simple ranked list.

## What to Look For

### Overlap of smells

- Cycles plus boundary leaks.
- God class plus wide fan-in.
- Duplication plus wide public API.

### Hot paths

Files touched often.
Modules used by many features.

### Blast radius

Many dependents.
Cross-context links.

### Safety hints

Tests exist and are active.
Clear owners in CODEOWNERS.

### Quick wins

Small surface with high overlap.
Low coupling but high smell density.

## Heuristics

- Count signals per module from 3.1 to 3.9.
- Weight by fan-in, churn, and layer rank.
- Prefer places with tests and owners.
- Flag risky zones with few tests.

## Output Rules

- One block per hotspot.
- Include Hotspot path, brief Why, Risk, Severity, Confidence.
- No fixes. No plans.
- Sort by Severity then Risk.
- Use exact module names and paths from earlier agents.

## Scoring Guide

- Start at 1 point per confirming agent.
- +1 if fan-in is high.
- +1 if file churn is high.
- +1 if on a hot path.
- -1 if tests are strong and active.

**Risk is:**
- Low at 0 to 1.
- Medium at 2 to 3.
- High at 4+.

**Severity is:**
- Major when 3 or more agents overlap or any cycle touches core flows.
