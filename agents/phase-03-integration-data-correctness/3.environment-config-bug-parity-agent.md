# Environment Config Bug - Parity Agent

## Goal

List config mismatches across environments. No edits. No execution.

## Method

- Read .env*, Docker files, compose files, manifests, lockfiles.
- Compare local, staging, and production values.
- Flag gaps, drift, and path issues.
- Group by file and service.

## What to Look For

### Missing required keys

Present in one env, absent in another.
Empty values where non-empty is needed.

### Version drift

Language runtime differs between envs.
Library or image tag mismatch.

### Path and casing errors

Windows vs Linux path separators.
Case-sensitive file names on Linux that differ from local.

### Port and network conflicts

Same port reused by two services.
Service name mismatch in compose links.

### Feature flags and mode toggles

Flags enabled in one env but not others.

### Secrets handling

Secrets in .env locally but in a secret store in prod.
Missing mount or reference.

### Build args vs env vars

ARG used at build time, ENV expected at runtime.

### Timezone and locale

Different TZ or locale causing date or sort issues.

### Storage and volumes

Volume paths differ. Data not persisted in one env.

### Health checks and startup order

Missing healthcheck or wrong dependency order.

## Expected Output Format

Readable. One line per finding. Grouped by file and service.

```
File: docker-compose.yml
  - Service: api
    Issue: Image tag 'node:20' locally, 'node:18' in production
    Confidence: High
    Severity: Major

File: .env.staging
  - Service: web
    Issue: Missing required key 'API_BASE_URL' present in .env.production
    Confidence: High
    Severity: Major

File: backend/.env.local
  - Service: worker
    Issue: QUEUE_NAME case differs from code reference 'queue_name'
    Confidence: Medium
    Severity: Major

File: k8s/deployment.yaml
  - Service: api
    Issue: ENV expects 'REDIS_URL' but only build ARG 'REDIS_URL' set in Dockerfile
    Confidence: High
    Severity: Major

File: scripts/migrate.sh
  - Service: db
    Issue: Uses Windows-style path 'C:\data' not valid in Linux container
    Confidence: High
    Severity: Moderate
```

## Output Rules

- List every mismatch you find.
- Include file, service, short issue text, confidence, and severity.
- Do not propose patches. Do not change code.
- Sort by file path, then by service, then by line if known.
- Use exact keys, versions, and paths seen in files.

## Severity

- **Major** for missing keys, version drift, wrong ports, or secret linkage that can break deploy.
- **Moderate** for path style, casing, or timezone differences that degrade behavior.

## Confidence

- **High** when values clearly differ across env files or manifests.
- **Medium** when inferred from naming or comments.
- **Low** only when intent is unclear.
