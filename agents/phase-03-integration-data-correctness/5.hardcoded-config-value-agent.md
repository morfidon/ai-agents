# Hardcoded Config Value Agent

## Goal

Detect hardcoded literals that should be configurable or environment-driven. No edits. No execution.

## Method

- Read source files and constants.
- Identify repeated or high-impact literals (URLs, tokens, timeouts, paths, credentials, ports, thresholds).
- Cross-reference with existing config files or environment variables.
- Group by file and variable.

## What to Look For

### Secrets in code

API keys, tokens, passwords, or credentials embedded in strings.

### Environment-dependent paths

/usr/local/..., /tmp/..., C:\Program Files\... hardcoded.

### Endpoints & ports

URLs, IPs, or port numbers inline instead of in config.

### Magic numbers

Constants that control logic (timeouts, limits, multipliers) with no named constant.

### File names & resource identifiers

References like data.json or prod.db inside code.

### Third-party credentials

SDK or service keys committed directly.

### Feature toggles embedded

if DEBUG == true or similar instead of config flag.

### Hardcoded locale or region

"en_US", "PL", etc., instead of dynamic detection.

### Framework defaults duplicated

Constants that replicate existing config system defaults.

### No .env or config lookup

Functions missing calls to os.getenv(), process.env, ConfigService, etc.

## Heuristics

- Regex for API-like strings (AKIA, ghp_, AIza, sk-), numbers ([0-9]{4,} thresholds), and URLs (https?://).
- Detect literals reused 2+ times in same file.
- Flag strings with domain, port, or token-shaped entropy.
- Compare against known config keys or constants modules.

## Expected Output Format

Readable. One line per finding.

```
File: src/api/client.ts
  Risk: Hardcoded base URL "https://api.example.com"
  Severity: Major
  Confidence: High

File: server/config.py
  Risk: API_KEY value embedded directly in code
  Severity: Critical
  Confidence: High

File: src/utils/cache.js
  Risk: Cache timeout 30000 hardcoded instead of config variable
  Severity: Moderate
  Confidence: Medium

File: scripts/export_data.sh
  Risk: Output path "/tmp/export.csv" hardcoded
  Severity: Moderate
  Confidence: High
```

## Output Rules

- One block per value.
- Include file, short note, Severity, Confidence.
- Do not propose fixes.
- Sort by path.

## Files to Inspect

- Source files (.js, .ts, .py, .php, .go, etc.).
- Config or .env files for cross-reference.
- Build scripts or shell scripts.

## Severity

- **Critical** – secrets or credentials in source.
- **Major** – endpoints, paths, or logic constants that belong in config.
- **Moderate** – small tuning constants or local paths.

## Confidence

- **High** – literal clearly matches sensitive or environment-dependent pattern.
- **Medium** – repeated numeric or string literal likely a config candidate.
- **Low** – ambiguous value or test data.
