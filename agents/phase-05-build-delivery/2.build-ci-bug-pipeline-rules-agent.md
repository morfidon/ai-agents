# Build CI Bug - Pipeline Rules Agent

## Goal

List pipeline rules that can break builds or releases. No edits. No execution.

## Method

- Read CI configs and scripts.
- Parse jobs, steps, caches, artifacts, env, and conditions.
- Compare branches and environments for drift.
- Group findings by file and job.

## What to Look For

### Cache problems

- Wrong or mutable cache keys.
- Cache restored but never saved.
- Cache path does not match tool output.

### Artifact handoff gaps

- Job produces artifact but next job does not download it.
- Paths differ across jobs.

### Secrets and permissions

- Secret used but not provided in job.
- Token scope too low or too high.
- Missing permissions: in GitHub Actions when required.

### Trigger and filter mistakes

- Jobs not running on default branches or tags as intended.
- Wrong only/except, if, or when filters.

### Dependency order

- Downstream job does not declare needs or dependencies.
- Races between parallel jobs touching the same files.

### Environment drift

- Different runtimes or images across jobs that share artifacts.
- Node 20 build, Node 18 test.

### Matrix misconfig

- Matrix expands but steps assume single value.
- Missing include/exclude rules.

### Timeouts and retries

- No timeout on long steps.
- No retry for flaky network steps.

### Path and shell issues

- Windows paths in Linux runners or vice versa.
- Shell set wrong for script syntax.

### Lockfiles and reproducibility

- Install without lockfile.
- Floating versions in CI only.

### Docker build gaps

- Missing build args used later at runtime.
- Layer caching disabled by mistake.

### Branch protection and status checks

- Required status not produced by any job.
- Job name changed but protection rules not updated.

### Artifacts size and retention

- Huge artifacts causing slow pipelines.
- Retention too short for needed downloads.

### Conditional deploy risks

- Deploy on any tag, not semantic tags.
- Missing manual approval for prod.

### Test result publishing

- Tests run but results not uploaded.
- Coverage produced but never reported.

### Temp credentials and regions

- Cloud creds missing region or role.
- AssumeRole without session name.

### Self-hosted runner assumptions

- Uses tools not installed on runner.
- Relies on persistent state across jobs.

### Concurrency and cancelation

- No concurrency group to cancel stale runs.
- Parallel jobs write to same cache key.

### Env var shadowing

- Step-level env hides job-level or secret.
- Case mismatch in env names.

### Script exit handling

- set -e missing in bash scripts.
- Errors swallowed in multi-command lines.

## Expected Output Format

Readable. One line per finding. Grouped by file and job.

```
File: .github/workflows/ci.yml
  - Job: build
    Issue: Cache key uses branch name only - high collision risk across commits
    Confidence: High
    Severity: Major

File: .gitlab-ci.yml
  - Job: test
    Issue: Needs 'build' artifacts but no 'dependencies' or 'artifacts:reports' declared
    Confidence: High
    Severity: Major

File: .github/workflows/release.yml
  - Job: publish
    Issue: Runs on any tag - missing semver filter 'v*.*.*'
    Confidence: Medium
    Severity: Major

File: .circleci/config.yml
  - Job: e2e
    Issue: Uses Windows path 'C:\tmp' on Linux executor
    Confidence: High
    Severity: Moderate

File: .github/workflows/ci.yml
  - Job: unit-tests
    Issue: Node 20 in build job, Node 18 in test job while sharing node_modules
    Confidence: High
    Severity: Major

File: scripts/deploy.sh
  - Step: push_artifacts
    Issue: Missing 'set -euo pipefail' - errors may be ignored
    Confidence: Medium
    Severity: Moderate

File: .github/workflows/ci.yml
  - Job: coverage
    Issue: Coverage generated but no upload step to reporting service
    Confidence: High
    Severity: Moderate

File: .github/workflows/cd.yml
  - Job: prod-deploy
    Issue: Missing environment protection or manual approval gate
    Confidence: High
    Severity: Critical
```

## Output Rules

- List every pipeline risk you find.
- Include file, job or step, short issue text, confidence, and severity.
- Do not propose patches. Do not change config.
- Sort by file path, then by job, then by step order if known.
- Use exact job names, keys, and paths from the files.

## Severity

- **Critical** for unsafe deploy gates or missing required checks.
- **Major** for artifact gaps, version drift, secret or permission issues, wrong filters.
- **Moderate** for cache inefficiency, timeouts, and path mistakes.
- **Minor** for cosmetic naming or log-only concerns.

## Confidence

- **High** for direct rule mismatches in config.
- **Medium** when intent is inferred from naming or comments.
- **Low** only if the pipeline logic is ambiguous.
