# Observability Bug - Telemetry Agent

## Goal

List spots where problems happen but you cannot see them. No edits. No execution.

## Method

- Read logging, tracing, and metrics code.
- Scan handlers, middleware, background jobs, and hot paths.
- Flag places that hide errors or lose context.
- Group by file and function or route.

## What to Look For

### Swallowed exceptions

- try blocks that catch and ignore.
- catch logs only message, not stack.

### Logs with no context

- No request ID, user ID, or trace ID.
- Plain strings with no structured fields.

### Missing key metrics

- No latency metric for hot endpoints.
- No error counter on failure branches.

### Silent retries

- Retries with backoff but no log or metric.

### Background jobs with no traces

- Workers do big work but emit nothing.

### Dropped correlation

- New threads or async tasks lose trace context.

### No sampling rules

- Traces never sampled for critical routes.

### PII in logs

- Sensitive data written to logs.

### Unbounded log volume

- Debug logs in hot loops.

### Health checks without signals

- No readiness or liveness logs or metrics.

## Expected Output Format

Readable. One line per finding. Grouped by file and function or route.

```
File: api/orders/controller.ts
  - Route: POST /orders
    Missing Telemetry: Catches error and returns 500 without logging stack or requestId
    Confidence: High
    Severity: Moderate

File: services/payments/worker.py
  - Function: process_refund()
    Missing Telemetry: No trace span or latency metric around external API call
    Confidence: Medium
    Severity: Moderate

File: web/middleware/tracing.go
  - Func: WithTrace
    Missing Telemetry: Spawns goroutine but drops parent trace context
    Confidence: High
    Severity: Moderate

File: src/common/log.ts
  - Function: logInfo()
    Missing Telemetry: Logs plain string without structured fields or userId
    Confidence: High
    Severity: Moderate

File: jobs/email_sender.rb
  - Job: EmailSender
    Missing Telemetry: Retries on failure with no counter or alert
    Confidence: Medium
    Severity: Moderate
```

## Output Rules

- List every observability gap.
- Include file, function or route, short note, confidence, and severity.
- Do not propose patches. Do not change code.
- Sort by file path, then by function or route, then by line if known.
- Use exact identifiers you see.

## Severity

- **Moderate** by default - users cannot trace issues, but app may still run.
- Raise to **Major** if gaps hide security or payment failures.

## Confidence

- **High** when exceptions are caught and not logged with stack or context.
- **Medium** when gaps are inferred from missing IDs or spans.
- **Low** only if intent is unclear.
