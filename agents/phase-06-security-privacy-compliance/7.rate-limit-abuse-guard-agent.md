Goal:Defend high-traffic or sensitive endpoints against abuse. No edits. No execution.

Method:

Read API gateway configs, middleware, and reverse proxies.

Read route handlers and job endpoints.

Check for rate-limit, throttle, and idempotency patterns.

Compare to security baseline or OWASP API 2023 guidance.

Group findings by route or service.

What to Look For:

No rate-limit or burst control

Routes with no throttle middleware or gateway quota.

Public APIs using free-form POST/PUT with unlimited hits.

Missing CAPTCHA or challenge

Login, signup, and password reset flows unprotected.

Lack of idempotency

No Idempotency-Key header handling on writes.

Payment or order endpoints may double-execute.

Shared limits across tenants

Same quota bucket for all tenants or users.

Heavy tenant can starve others.

Static burst size

Hardcoded burst window ignoring dynamic load or tier.

No 429 response policy

Over-limit requests return 500 or hang instead of 429.

Rate limits not enforced per auth context

Based only on IP, not on token or account.

Background or async jobs

Webhooks or queue consumers unbounded, allowing floods.

Bot detection missing

No UA filtering, signature, or behavioral checks.

Abuse-prone routes

/auth/login, /forgot, /invite, /public/search wide open.

Quota bypass

Multiple endpoints hit same backend but only one is limited.

Tiering drift

Premium and free tiers use same quota config.

API key reuse

Shared API keys among services break per-client rate logic.

No cooldown or penalty

Repeated failed logins never trigger delays.

Reverse proxy misconfig

X-Forwarded-For not honored or trusted improperly, making IP limits useless.

Heuristics:

Express/FastAPI/Django: look for @limiter.limit, SlowAPI, ratelimit(), or middleware absent.

NGINX/Envoy: inspect limit_req_zone, rate_limits, burst, nodelay.

Cloud APIs: check API Gateway/CloudFront/WAF rules.

Search code for Idempotency-Key, Retry-After, X-RateLimit-*.

Logs: repeated 5xx or duplicates hint missing limit.

Include route or file, short note, Severity, Confidence.

Do not suggest fixes.

Sort by route or service path.

Use exact endpoints or filenames from code.

Files to Inspect:

API gateway configs.

Server middleware and route definitions.

NGINX/Envoy/Traefik rules.

Worker and webhook consumers.

Security docs or tier configs.

Severity:

Critical for idempotency gaps on payments or destructive actions.

Major for missing rate limits, CAPTCHA, or per-tenant quotas.

Moderate for weak thresholds or non-standard responses.

Confidence:

High when explicit limit code or headers missing.

Medium when inferred from route context.

Low only if rate-limit handled outside repo.
