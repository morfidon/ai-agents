Goal:List places where feature flags do not match across code and config. No edits. No execution.

Method:

Read flag definitions.

Read all usage sites.

Read defaults in env files and configs.

Compare names, defaults, and scopes.

Group by file and flag.

What to Look For:

Undefined flags

Code reads a flag that is not defined anywhere.

Inverted defaults

Default true in code, default false in env.

Staging differs from production without notes.

Dead flags

Flag never read. Only defined.

Flag read but always the same branch taken.

Retired flags still in code

Comments or docs say removed, but code still checks it.

Name drift

FEATURE_NEW_CHECKOUT vs FEATURE_CHECKOUT_NEW.

Case changes between files.

Scope mismatch

Backend flag missing in frontend.

Mobile uses build-time flag, server expects runtime flag.

Type mismatch

String "true" vs boolean true.

Number thresholds parsed as strings.

Missing fallback

Code uses flag value with no safe default.

Rollout rules missing

Percent rollout defined but code treats as boolean.

Env-specific drift

Flag on in dev by default but off in prod with no override path.

Expected Output Format:Readable. One line per finding. Grouped by file and flag.

File: web/src/flags.ts
  - Flag: FEATURE_NEW_CHECKOUT
    Inconsistency: Used in code but not defined in flags catalog
    Confidence: High
    Severity: Major

File: config/.env.staging
  - Flag: FEATURE_SEARCH_V2
    Inconsistency: Default=true here, default=false in .env.production
    Confidence: High
    Severity: Major

File: services/api/flags.yaml
  - Flag: ENABLE_COUPONS
    Inconsistency: Defined but never referenced in codebase
    Confidence: High
    Severity: Moderate

File: mobile/app/build.gradle
  - Flag: PAYWALL_ENABLED
    Inconsistency: Build-time constant on mobile, server expects runtime toggle
    Confidence: Medium
    Severity: Major

File: server/src/checkout.ts
  - Flag: FEATURE_NEW_CHECKOUT
    Inconsistency: Name differs from frontend 'FEATURE_CHECKOUT_NEW'
    Confidence: High
    Severity: Major


Output Rules:

List every flag inconsistency.

Include file, flag name, short inconsistency text, confidence, and severity.

Do not propose patches. Do not change code or config.

Sort by file path, then by flag name.

Use exact flag names seen in files.

Severity:

Major for undefined flags, inverted defaults across envs, scope mismatches, and name drift that breaks toggling.

Moderate for dead or retired flags and missing rollouts.

Confidence:

High when a flag is referenced with no definition or exact names differ.

Medium when behavior is inferred from comments or environment notes.

Low only if intent is unclear.
